Executive Summary

1. Executive Summary

Veduwa is an AI-powered IT job marketplace that automates the hiring pipeline from job posting to candidate screening. The platform uses vector embeddings for semantic matching, Claude AI for real-time candidate screening via streaming chat, and a modern full-stack architecture built for production scale.

2. System Architecture

Veduwa follows a decoupled architecture with a Next.js frontend deployed on Vercel's edge network, a FastAPI backend on Render, and Supabase providing both PostgreSQL and authentication services.

AI Pipeline

6. AI Pipeline

All AI operations use the Anthropic Claude API exclusively. No OpenAI dependency. The pipeline processes jobs and candidates through 5 stages.

Job Description ──► JD Parser (Haiku) ──► Structured JSON │ Resume PDF ────────► Resume Parser (Haiku) ────►│ ▼ Embedding Generator (384-dim vectors) │ ▼ Match Scorer (cosine + skills) │ ▼ AI Screening Chat (Sonnet streaming) │ ▼ Summary Generator (Sonnet)

AI Services

ServiceModelPurposeOutput
JD Parserclaude-haiku-4-5Extract skills, seniority, domainJSON
Resume Parserclaude-haiku-4-5Extract skills, experience, educationJSON
EmbeddingsHash-based (384d)Semantic vector generationfloat[384]
Match ScorerNumPyCosine similarity + skills overlap0–100 score
Screening Chatclaude-sonnet-4Real-time interviewSSE stream
Summarizerclaude-sonnet-4Employer assessmentText

Match Score Formula

match_score = (cosine_similarity × 0.4 + skills_overlap_ratio × 0.6) × 100

  • Cosine similarity (40%) — Semantic similarity between job and candidate embedding vectors
  • Skills overlap (60%) — Ratio of matched skills to required skills, case-insensitive

Data Flow Pattern

Authentication & Authorization

8. Authentication & Authorization

Auth Flow

1. User signs in via Supabase Auth (email/password or Google OAuth) 2. Supabase returns JWT access token 3. Frontend stores token in Supabase session (httpOnly cookie) 4. API requests include: Authorization: Bearer <token> 5. Backend verifies token via Supabase Auth API 6. User record auto-created in DB on first login 7. Role extracted from user_metadata.role ('employer' | 'candidate')

Performance & Security

12. Performance & Security

Security Measures

  • JWT-based auth via Supabase
  • Role-based access control (RBAC)
  • CORS whitelist configuration
  • No secrets in client-side code
  • Environment variables for credentials
  • SQL injection prevention (SQLAlchemy ORM)
  • Input validation (Pydantic v2)
  • HTTPS enforced on all endpoints

Performance Features

  • Async database operations (asyncpg)
  • Connection pooling (Supabase pooler)
  • Static page generation (Next.js SSG)
  • Lazy-loaded API data with fallback
  • GPU-accelerated animations (Framer)
  • Image optimization (next/image)
  • Code splitting per route
  • Edge deployment (Vercel CDN)
